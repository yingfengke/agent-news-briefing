──────────────────────────────────────────────────────────────
一、如何获取 QQ 邮箱授权码
──────────────────────────────────────────────────────────────

QQ邮箱的「授权码」不同于你的QQ密码，需要单独生成：

  1. 打开浏览器 → 登录 QQ邮箱 (mail.qq.com)
  2. 点击顶部「设置」→ 选择「账户」选项卡
  3. 往下翻找到「POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV服务」
  4. 找到「SMTP服务」行，点击右侧「开启」（如果已开启则点「生成授权码」）
  5. 按提示发送短信到指定号码
  6. 系统会生成一串 16 位字母授权码（类似 tnvurrlyyihtdggf）
  7. 复制并保存好，这就是 .env 里的 AUTH_CODE


──────────────────────────────────────────────────────────────
二、如何填写 .env 文件
──────────────────────────────────────────────────────────────

项目根目录的 .env 文件内容如下（已经填好你的配置）：

┌─────────────────────────────────────────────────────────┐
│ # === 硅基流动 API（所有 AI 分析强制使用此配置） ===    │
│ API_BASE_URL=https://api.siliconflow.cn                  │
│ API_KEY=sk-你的密钥                                      │
│ MODEL_NAME=deepseek-ai/DeepSeek-V4-Flash                 │
│                                                          │
│ # === QQ邮箱 SMTP 配置（用于每天自动发送简报） ===       │
│ SENDER_EMAIL=你的QQ号@qq.com                             │
│ AUTH_CODE=你的16位授权码                                  │
│ RECEIVER_EMAIL=收件邮箱地址                               │
│ SMTP_SERVER=smtp.qq.com                                  │
│ SMTP_PORT=465                                            │
└─────────────────────────────────────────────────────────┘

各字段说明：

  API_BASE_URL   硅基流动 API 地址，一般不用改
  API_KEY        你的硅基流动 API 密钥（以 sk- 开头）
  MODEL_NAME     模型名称，默认 DeepSeek-V4-Flash

  SENDER_EMAIL   发件人邮箱（你的QQ邮箱完整地址）
  AUTH_CODE      QQ邮箱授权码（不是QQ密码！见第一部分）
  RECEIVER_EMAIL 收件人邮箱（可以填自己的手机邮箱）
  SMTP_SERVER    QQ邮箱SMTP服务器，一般不用改
  SMTP_PORT      SSL端口465，一般不用改


──────────────────────────────────────────────────────────────
三、如何本地测试运行
──────────────────────────────────────────────────────────────

第一次使用需要安装依赖（只需一次）：

  pip install python-dotenv

然后按顺序测试：

  ★ 测试 RSS 抓取 + AI 分析：
    python generate_briefing.py

    正常输出：
    - 显示 5 个 RSS 源各抓取 3 条新闻
    - 显示 AI 分析成功（返回 5 条筛选结果）
    - 显示"[成功] 已更新 5 条新闻到 HTML"

  ★ 测试邮件发送：
    python send_email.py

    正常输出：
    - "连接 SMTP 服务器 smtp.qq.com:465 ... ✔ 发送成功"

  ★ 一次性跑完整个流程：
    python generate_briefing.py && python send_email.py

  ★ 查看生成效果：
    用浏览器打开 tech-briefing.html 即可看到完整的简报页面


──────────────────────────────────────────────────────────────
四、⚠️ 重要安全警告（必读！）
──────────────────────────────────────────────────────────────

  ⚠️  ⚠️  ⚠️  ⚠️  ⚠️  ⚠️  ⚠️  ⚠️  ⚠️  ⚠️  ⚠️  ⚠️

  .env 文件里包含你的：
    • 硅基流动 API Key（可以调用大模型，有费用）
    • QQ邮箱授权码（别人拿到可以发邮件冒充你）

  这两个是敏感凭证，如果泄露到网上，后果严重！

  请务必遵守以下规则：

  ① .env 文件已经加入了 .gitignore，git 不会把它提交上去
  ② 永远不要手动把 .env 的内容贴到 GitHub Issues、PR 或公开论坛
  ③ 如果误把 .env 提交到了 GitHub，立即去：
     - 硅基流动官网 重置 API Key
     - QQ邮箱 → 设置 → 账户 → 关闭并重新生成授权码
  ④ 如果你把代码推送到 GitHub，需要在仓库的 Settings → 
     Secrets and variables → Actions 中手动添加以下 Secrets：
     API_BASE_URL、API_KEY、MODEL_NAME、SENDER_EMAIL、
     AUTH_CODE、RECEIVER_EMAIL、SMTP_SERVER、SMTP_PORT
     （请把 .env 里的值逐个复制到 GitHub Secrets）
